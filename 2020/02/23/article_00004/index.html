eclipse、idea、notepad++设置编码为UTF-8，换行符为Unix

 2020-02-23  2020-03-08
阿里编码规约
3 分钟 读完 (大约 524 个字) 访问量：0

根据阿里编码规约，将eclipse、idea、notepad++的编码设置为UTF-8，换行符设置为Unix，有利于编码；

eclipse、idea、notepad++设置编码为UTF-8，换行符为Unix

环境说明：

  • eclipse版本：Photon Release (4.8.0)
  • idea版本：2018.03.06
  • notepad++版本：v7.8.4

eclipse中的设置

  1. Window -> Preferences，进入参数设置页面；

  2. Preferences页面左侧General->Workspaces，右侧Text file encoding选项设置为Other:UTF-8（文本文件编码），New text file line delimiter选项设置为Other:Unix（换行符格式），点击右下角Apply；

  3. Preferences页面左侧General->Content Types，右侧选中Text，将底部的Default encoding选项设置为UTF-8，并点击Update；

  4. 还是3中的路径，展开Text（就是3中设置的Text）项，选中Java Properties File选项，将底部的ISO-8859-1更换为UTF-8，并点击右侧Update；

  5. Preferences页面左侧Web->JSP Files，右侧Encoding项设置为ISO 10646/Unicode(UTF-8)，点击右下角Apply；

idea中的设置

  1. File -> Setting…，进入设置页面；

  2. 左侧Editor->File Encodings，右侧上部Global Encoding选项和Project Encoding选项设置为UTF-8，下部Default encoding for properties files选项设置为UTF-8，并勾选后面Transparent native-to-ascii conversion复选框，Create UTF-8 files选项设置为with NO BOM，点击右下角Apply，UTF-8编码设置完成；

  3. 设置页面左侧Editor->Code Style，右侧Line separator选项设置为Unix and OS X(\n)，点击右下角Apply，Unix格式换行符设置完成；

notepad++中的设置

  1. 设置->首选项；

  2. 左侧“新建”，右侧“格式（行尾）”选项设置为“Unix(LF)”，“编码”选项设置为“UTF-8(无BOM)”，并勾选“适用于打开ANSI文件”；
